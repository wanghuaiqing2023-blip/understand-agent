# AGENTS.md

本文件是 understand-agent 项目的项目级长期记忆。Codex 在进入本项目工作时，应优先读取并遵守这里的协作规则。

## 项目定位

understand-agent 的目标是逐步构建一个通用智能体：从最小任务执行闭环开始，持续增加规划、工具调用、记忆、评估、工程协作、多智能体协作等能力，最终逼近 Codex、OpenClaw 等工程型智能体的核心能力。

本项目不是单纯堆功能，而是同时建设一套关于智能体的概念体系、战略判断框架和长期认知资产。

## 核心哲学

智能体负责加速执行，人类必须负责概念掌控。

代码能力的增长如果没有伴随概念澄清，会导致人类逐渐失去对项目的战略主导。项目推进时，必须同时关注“能做什么”和“我们如何理解它”。

本项目以实现需求为根本驱动力。概念澄清、文档沉淀和技术决策都必须服务于真实需求的识别、实现、验证和演化，不能脱离软件项目的交付本质。

两条主线必须同步推进：

```text
代码进化线：功能、模块、测试、工具、接口
认知进化线：概念、原则、战略、边界、判断框架
```

## 基本原则

1. 代码记录结果，文档记录认知。
2. 能力增长必须伴随概念澄清。
3. 不把短期执行效率凌驾于长期可理解性之上。
4. 不把 Codex 的自动记忆当作项目唯一记忆；关键规则必须写入 `AGENTS.md` 或 `docs/`。
5. 人类不需要掌握每个实现细节，但必须掌握能力结构、系统边界、核心概念和战略路线。
6. 项目推进以需求为根本驱动力；新增能力必须能对应到明确需求、验收标准或阶段目标。

## 文档纪律

每次重要推进都要考虑是否更新文档：

- 新增重要能力：更新 `docs/01-requirements/`。
- 形成关键概念：更新 `docs/02-concepts/`。
- 做出技术取舍：新增或更新 `docs/03-decisions/` 中的 ADR。
- 完成阶段性工作：更新 `docs/04-dev-log/`。
- 发生认知升级：新增 `docs/05-breakthroughs/` 突破笔记。

当前文档体系：

```text
docs/
  00-vision/          项目愿景、路线图、长期方向
  01-requirements/    需求池、里程碑、能力分层
  02-concepts/        智能体关键概念与理解框架
  03-decisions/       架构决策记录 ADR
  04-dev-log/         开发日志、阶段复盘
  05-breakthroughs/   认知突破、概念升级记录
```

## 需求来源

本项目优先从真实任务和专业评测中收集需求。专业智能体评测基准可以作为需求矿场，例如 Terminal-Bench、SWE-bench、AgentBench、WebArena、OSWorld、GAIA 等。

使用评测基准时必须做需求降级：不要直接追逐完整 benchmark 或成熟智能体的高阶任务，而是抽取其中的能力类型、任务结构、验收方式和失败模式，转化为当前阶段可实现、可验证的小需求。

需求来源优先级：

```text
本项目狗粮任务
-> Codex / OpenCode 行为观察降级需求
-> 专业评测基准降级需求
-> 自建基础任务集
-> 失败案例
-> 成熟项目 issue 降级需求
```

## 新增能力时必须回答的问题

每新增一个重要能力，都要至少思考这些问题：

- 这个能力改变了我们对智能体的哪个理解？
- 它属于感知、规划、执行、记忆、评估，还是协作？
- 它增强的是智能体的自主性，还是可控性？
- 它会带来哪些新的风险？
- 人类需要掌握哪个新概念，才能继续控制项目？
- 是否需要新增需求、ADR、概念文档或认知突破记录？

## 工程协作规则

- 优先尊重仓库现有结构和文档体系。
- 修改代码前先理解相关文件和项目意图。
- 变更应尽量小而清晰，避免无关重构。
- 重要实现应配套验证方式；无法验证时要明确说明。
- 不覆盖用户已有修改，不擅自回滚不属于自己的改动。
- 涉及破坏性操作、外部网络、系统级修改或依赖安装时，应先取得用户确认。

## 技术路线倾向

早期可以优先使用 Python 或 Node.js 实现最小智能体，但工具协议必须保持语言无关。

- Python 适合快速验证、AI 生态、数据处理、文档处理和本地自动化。
- Node.js / TypeScript 适合异步编排、前端通信、浏览器自动化、插件生态和产品化。
- 智能体核心应被设计为任务调度、工具系统、状态系统和反馈循环，而不是绑定到某一种语言。

## 记忆边界

Codex 的自动 Memories 可以作为辅助回忆层，但本项目的正式长期记忆分三层：

```text
~/.codex/memories/     Codex 自动回忆，不作为主要控制面
AGENTS.md             项目级长期工作规则
_docs/ 或 docs/        人类和项目共同使用的长期认知资产
```

本项目以 `AGENTS.md` 和 `docs/` 为可审查、可维护、可传承的正式记忆。
