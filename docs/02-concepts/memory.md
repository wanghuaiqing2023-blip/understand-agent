# 概念：记忆

## 当前定义

记忆是智能体跨任务保留信息、偏好、经验和项目知识的机制。

## 记忆不等于上下文

上下文通常是一次对话或一次任务内临时可见的信息。

记忆是跨任务、跨会话、可持续积累的信息。

## 记忆类型

- 项目事实：项目名称、目录结构、技术栈
- 用户偏好：沟通风格、技术偏好、工作习惯
- 决策历史：为什么选择某个方案
- 失败经验：哪些方法试过但不合适
- 概念沉淀：关键理解、原则、判断框架

## 风险

- 错误记忆会长期污染判断
- 过度记忆会降低灵活性
- 未经用户确认的记忆可能侵犯边界
- 把所有东西塞进 prompt 会导致上下文混乱

## 初始设计倾向

第一阶段不急于做复杂记忆系统。先用文档和本地文件沉淀项目事实、决策和认知突破，让人类可以直接阅读和修正。
