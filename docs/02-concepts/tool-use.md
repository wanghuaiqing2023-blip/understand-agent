# 概念：工具调用

## 当前定义

工具调用是智能体把语言意图转换为外部行动的机制。

LLM 本身主要生成文本和结构化指令，工具系统负责把这些指令变成真实世界中的操作。

## 工具调用的基本结构

```text
工具名称
工具描述
输入 schema
执行函数
输出结果
错误信息
权限边界
```

## 关键价值

- 让智能体能操作文件、命令、浏览器和外部 API
- 让复杂任务可以拆成可验证的动作
- 让行动过程可记录、可复盘、可限制

## 设计问题

- 工具描述如何让模型准确选择？
- 输入 schema 如何避免歧义？
- 工具执行失败时如何反馈？
- 哪些工具需要人类确认？
- 工具是否应该支持多语言实现？

## 初始设计倾向

工具协议应尽量语言无关。即使第一版使用 Python 或 Node.js 实现，也应该允许未来通过子进程、HTTP、MCP 或 RPC 接入其他语言工具。
