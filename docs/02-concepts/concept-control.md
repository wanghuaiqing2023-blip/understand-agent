# 概念掌控

## 核心判断

智能体负责加速执行，人类必须负责概念掌控。

Codex 这类智能体可以快速完成局部任务，但它不天然拥有项目责任感、历史连续性、战略取舍和价值判断。

## 为什么需要概念掌控

当智能体能力越来越强，代码增长会变得很快。如果人类的概念理解没有同步增长，项目会出现一种危险状态：

```text
代码能力越来越强
人的理解越来越滞后
项目表面在进步
控制权实际从理解滑向依赖
```

## CEO 式项目理解

CEO 不需要掌握公司里的每一个细节，但必须掌握：

- 公司处在什么阶段
- 核心能力是什么
- 关键风险是什么
- 哪些事情是战略重点
- 哪些事情只是局部优化
- 当前增长是否仍服务长期目标

智能体项目也是如此。人类不必理解每个实现细节，但必须理解能力结构、系统边界、核心概念和战略路线。

## 原则 002：能力增长必须伴随概念澄清

每新增一个重要能力，我们都要回答：

- 这个能力改变了我们对智能体的哪个理解？
- 它属于感知、规划、执行、记忆、评估，还是协作？
- 它增强的是智能体的自主性，还是可控性？
- 它会带来哪些新的风险？
- 人类需要掌握哪个新概念，才能继续控制项目？

## 项目中的落地方式

- 功能需求写入 `docs/01-requirements/`
- 关键概念写入 `docs/02-concepts/`
- 技术取舍写入 `docs/03-decisions/`
- 每日或阶段进展写入 `docs/04-dev-log/`
- 认知突破写入 `docs/05-breakthroughs/`
