# 概念：评估

## 当前定义

评估是判断智能体能力是否真实提升的机制。

## 为什么需要评估

智能体很容易表现得“像是会了”，但真实能力需要通过任务结果验证。

如果没有评估，项目会陷入：

- 功能越来越多
- 演示越来越华丽
- 真实成功率不清楚
- 失败模式无法复盘

## 评估维度

- 完成率：任务是否完成
- 正确性：结果是否符合预期
- 稳定性：多次执行是否可靠
- 可解释性：人类是否理解它做了什么
- 可控性：人类是否能中断、审查、回滚
- 成本：时间、token、工具调用、系统资源

## 初始评估方式

早期采用简单任务集：

- 文件读取任务
- 文件修改任务
- 搜索定位任务
- 命令执行任务
- 小型代码修复任务

每个任务记录输入、期望结果、实际结果、失败原因和改进建议。
