# 概念：规划

## 当前定义

规划是智能体把用户目标拆成可执行步骤，并在执行中根据反馈调整路径的能力。

## 最小规划

最小规划不追求复杂，而追求清楚：

- 当前目标是什么
- 下一步做什么
- 为什么做这一步
- 什么情况表示完成

## 规划与执行的关系

规划不能脱离执行。智能体需要在执行中观察结果，并根据新信息调整计划。

```text
计划
  -> 执行
  -> 观察
  -> 修正计划
  -> 继续执行
```

## 关键问题

- 什么任务需要显式计划？
- 什么任务可以直接执行？
- 计划应该暴露给用户多少？
- 计划失败时如何重新规划？
- 如何避免过度规划导致行动迟缓？

## 项目原则

早期规划应简单、透明、可被人类理解。复杂任务再逐步引入状态机、依赖图和长期任务调度。
