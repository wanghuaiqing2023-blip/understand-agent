# 智能体项目文档总览

这个目录用于沉淀智能体项目的代码演化、需求演化和概念演化。

我们的核心原则：

> 代码记录结果，文档记录认知。

## 目录结构

```text
docs/
  00-vision/          项目愿景、路线图、长期方向
  01-requirements/    需求池、里程碑、能力分层
  02-concepts/        智能体关键概念与理解框架
  03-decisions/       架构决策记录 ADR
  04-dev-log/         开发日志、阶段复盘
  05-breakthroughs/   认知突破、概念升级记录
```

## 文档纪律

- 每增加一个重要能力，更新需求文档。
- 每做一个关键技术选择，新增或更新 ADR。
- 每出现一次认知升级，沉淀为突破笔记。
- 每完成一个阶段，写一次开发复盘。
- 能力增长必须伴随概念澄清。
