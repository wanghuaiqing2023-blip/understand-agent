# ADR-0002：工具协议应保持语言无关

## 状态

proposed

## 背景

智能体需要调用多种工具，包括文件系统、命令行、浏览器、文档处理、外部 API 和未来的插件。不同工具可能适合不同语言实现。

## 决策

工具协议保持语言无关。每个工具至少应描述：

- name
- description
- input_schema
- permission_level
- run(args)
- result
- error

## 原因

语言无关协议可以让项目早期快速实现，后期自然扩展到 Python、Node.js、Shell、HTTP 服务、MCP 服务或其他执行器。

这也能让智能体核心更像一个调度系统，而不是某一种语言里的函数集合。

## 替代方案

- 直接绑定 Python class：实现简单，但未来跨语言困难。
- 直接绑定 TypeScript interface：适合产品化，但对 Python 工具不够自然。
- 直接采用完整 MCP：标准化更强，但早期可能偏重。

## 后果

早期需要多写一些 schema 和协议约束，但长期能降低工具扩展成本。

## 复审条件

当工具数量超过 10 个，或出现跨进程、跨语言、远程工具时，需要复审协议是否要兼容 MCP 或其他标准协议。
