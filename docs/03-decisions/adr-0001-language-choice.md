# ADR-0001：早期语言选择

## 状态

proposed

## 背景

智能体项目可以使用 Node.js、Python、Rust、Go 等多种语言实现。当前我们需要为早期版本选择一个起步方向，同时避免把未来架构锁死。

## 决策

早期可以优先使用 Python 或 Node.js 实现最小智能体，但工具协议必须保持语言无关。

如果以学习智能体原理、快速验证能力为主，Python 是合理起点。如果以 Web 产品、插件、浏览器自动化和前端生态为主，Node.js / TypeScript 更顺手。

## 原因

Python 适合快速验证、AI 生态、数据处理、文档处理和本地自动化。

Node.js / TypeScript 适合异步编排、前端通信、浏览器自动化、插件生态和产品化。

智能体的本质是任务调度、工具系统、状态系统和反馈循环，不应被某一种语言绑定。

## 替代方案

- 全部使用 Node.js：产品化优势更明显，但研究和数据处理能力不如 Python 直接。
- 全部使用 Python：学习和实验速度快，但前端、桌面、浏览器扩展生态较弱。
- 一开始就多语言：架构更开放，但早期复杂度过高。

## 后果

早期实现可以快速推进，但必须从第一天开始约束工具接口，避免未来迁移成本过高。

## 复审条件

当项目开始做前端界面、浏览器插件、桌面应用或多人协作平台时，需要重新审视是否引入 TypeScript / Node.js 作为主控或界面层。
