# 开发日志：v0.1-alpha 工具注册系统

## 日期

2026-05-10

## 今天做了什么

- 建立 `understand_agent` Python 包。
- 实现 `ToolSpec`、`ToolResult`、`ToolContext` 和 `ToolRegistry`。
- 实现 3 个只读工具：`list_files`、`read_file`、`search_text`。
- 实现 CLI：`python -m understand_agent tools` 和 `python -m understand_agent call ...`。
- 添加标准库 `unittest` 单元测试和集成测试。
- 添加 GitHub Actions，在 push / PR 时自动运行测试。

## 遇到的问题

- Windows / 沙箱环境不适合在系统临时目录中动态创建测试文件，改用仓库内静态 fixture。
- Windows 默认控制台编码可能无法直接输出中文 JSON，CLI 显式将 stdout 配置为 UTF-8。

## 关键决策

- v0.1-alpha 先实现工具注册系统，而不是直接做完整自然语言 Agent。
- 第一版使用 Python 3.13、顶层包结构、零外部依赖和标准库 `unittest`。
- 内部 `ToolSpec` 保持模型无关，不绑定 OpenAI API 的 tool type。

## 概念收获

- 工具不是裸函数，而是带有描述、输入、权限和结果协议的行动能力。
- 工具注册系统是智能体从聊天程序走向行动系统的第一块骨架。
- 自动化测试应尽早进入项目，帮助需求变成可验证能力。

## 下一步

- 增加任务输入和最小计划生成。
- 评估是否为工具参数增加更明确的 schema 校验。
