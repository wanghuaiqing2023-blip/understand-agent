# 项目愿景

## 一句话愿景

构建一个从简单任务执行逐步进化到通用工程协作能力的智能体，同时沉淀一套人类可理解、可掌控、可持续演化的智能体思想体系。

## 项目定位

这个项目不是单纯复制某个现有智能体，而是在持续协作中逐步理解并实现智能体的核心能力：

- 任务理解
- 计划生成
- 工具调用
- 文件读写
- 命令执行
- 代码修改
- 测试与修复
- 记忆与上下文管理
- 浏览器与外部系统操作
- 多智能体协作
- 自我评估与持续改进

## 核心哲学

智能体负责加速执行，人类必须负责概念掌控。

人类不需要掌握每一行代码，但必须掌握项目发展的概念结构、战略节奏、边界条件和关键取舍。

## 双线演化

```text
代码进化线：功能、模块、测试、工具、接口
认知进化线：概念、原则、战略、边界、判断框架
```

两条线必须同步推进。代码能力的增长如果没有伴随概念澄清，就会导致人类逐渐失去对项目的战略掌控。

## 长期目标

最终智能体应逐步具备不亚于 Codex、OpenClaw 等工程型智能体的核心能力：

- 理解复杂任务
- 操作真实代码库
- 执行多步骤工作流
- 调用多种工具
- 处理失败和不确定性
- 与人类长期协作
- 把项目经验沉淀为可复用知识
