# 能力路线图

## 阶段 1：最小智能体

目标：让智能体能够完成一个简单任务闭环。

- 接收用户任务
- 生成简短计划
- 读取当前目录文件
- 搜索代码或文本
- 修改文件
- 执行命令
- 输出完成摘要

## 阶段 2：任务工作流

目标：让智能体从一次性执行变成可追踪的任务系统。

- 任务状态管理：pending / running / blocked / done
- 多步骤计划
- 进度记录
- 失败重试
- 工具调用日志
- 简单上下文压缩

## 阶段 3：工程能力

目标：让智能体具备真实代码库协作能力。

- 代码库理解
- 自动定位相关文件
- 自动修改代码
- 运行测试
- 根据报错继续修复
- 生成变更摘要
- 支持 Git 分支、提交、PR 流程

## 阶段 4：通用能力

目标：让智能体从代码助手扩展为通用任务执行系统。

- 浏览器操作
- 文档处理
- 表格处理
- PPT 处理
- 外部 API 接入
- 长期记忆
- 用户偏好学习
- 多项目工作区

## 阶段 5：高级智能体

目标：让智能体具备更强的自主性、协作性和评估能力。

- 多智能体协作
- 自我评估
- 自动生成需求列表
- 自动生成测试和基准
- 插件系统
- 权限与安全沙箱
- 可视化任务面板

## 路线图维护规则

- 新能力必须放入明确阶段。
- 如果一个能力跨越多个阶段，需要拆成更小能力。
- 每个阶段都要同时记录技术目标和概念目标。
